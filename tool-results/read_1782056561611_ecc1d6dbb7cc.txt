   595→   595→Task ID: FIX-CSS (critical recovery)
   596→   596→Agent: orchestrator (Z.ai Code)
   597→   597→Task: Fix broken preview — CSS compilation error preventing the app from loading.
   598→   598→
   599→   599→Root Cause:
   600→   600→- Tailwind v4.1.18 registers `aeon` as a color namespace when `--aeon-*` custom properties
   601→   601→  exist in `:root` (or when `text-[var(--aeon-core)]` arbitrary-value classes are used).
   602→   602→- When generating `shadow-lg`/`shadow-md` utilities, Tailwind v4 emits
   603→   603→  `--tw-shadow: 0 0 14px -3px var(--tw-shadow-color, var(--aeon-*))` where `*` is the
   604→   604→  "opaque" namespace placeholder.
   605→   605→- Lightning CSS (Turbopack's CSS parser) cannot parse the literal `*` in `var(--aeon-*)`,
   606→   606→  throwing `Unexpected token Delim('*')` → HTTP 500 on every page.
   607→   607→
   608→   608→Fix Applied:
   609→   609→1. Renamed ALL `--aeon-*` CSS custom properties to `--a_*` (underscore prefix breaks the
   610→   610→   Tailwind v4 namespace detection pattern `--<namespace>-<key>`).
   611→   611→   - `:root` block in globals.css: `--aeon-core` → `--a_core`, etc. (8 variables)
   612→   612→   - All 384 `var(--aeon-*)` references in .tsx/.ts files → `var(--a_*)`
   613→   613→   - All `[var(--aeon-*)]` arbitrary-value className strings → `[oklch(...)]` values
   614→   614→   - `src/lib/aeon.ts` cssVar fields → `--a_*`
   615→   615→   - `src/app/api/aeon/seed/route.ts` agent colors → `var(--a_*)`
   616→   616→2. Switched dev server from Turbopack to Webpack (`--webpack` flag in package.json dev
   617→   617→   script) as a belt-and-suspenders fix — Webpack's CSS pipeline doesn't use Lightning CSS
   618→   618→   and handles edge cases more gracefully.
   619→   619→3. Restored `--color-aeon-*` entries in `@theme inline` (now mapped to `--a_*`).
   620→   620→4. Restored the full globals.css with all HUD utilities, animations, and scrollbar styles.
